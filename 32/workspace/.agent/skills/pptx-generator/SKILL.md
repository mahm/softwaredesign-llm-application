# arXiv論文スライド生成スキル

## 概要

arXiv論文のURLからプレゼンテーションスライドを作成する。論文取得からPPTX生成まで、すべての手順をこのスキルに従って実行する。

## ステップ1: 論文の取得と分析

1. arXiv IDをURLから抽出する(例: 2301.00001)
2. `curl -sL https://arxiv.org/abs/{id}` でアブストラクトページを取得する
3. `curl -sL https://arxiv.org/html/{id}` でHTML本文を取得する
4. HTML版が存在しない場合(PDF限定の論文)はユーザーに伝え、対象外とする
5. HTMLタグからセクション構造(h2, h3)を読み取る
6. 数式はTeX表記のまま抽出し、図表の参照はテキストで要約する
7. タイトル、著者、アブストラクト、各セクションの内容を抽出する

## ステップ2: アウトラインの作成

- 標準では5枚のスライド構成を提案する(タイトルスライドは枚数に含めない)
- タイトルスライド + 背景と課題、提案手法、実験結果、考察、まとめの流れを基本とする
- ユーザーが枚数を指定した場合はそれに従う(例: 「3枚で」ならタイトル + 3枚 = 計4枚)
- 各スライドのタイトルと箇条書きの要点を示す
- ユーザーの確認が得られるまでアウトラインの修正を続ける

## ステップ3: スライドデータの準備と生成

アウトラインが確定したら、以下のJSON形式でデータを整理し`generate_pptx`ツールを1回呼び出す。

```json
{
  "title": "プレゼンテーションタイトル",
  "author": "著者名",
  "slides": [
    {
      "type": "title",
      "title": "メインタイトル",
      "subtitle": "サブタイトル"
    },
    {
      "type": "content",
      "title": "セクションタイトル",
      "bullets": ["項目1", "項目2", "項目3"]
    },
    {
      "type": "section",
      "title": "セクション区切り"
    }
  ]
}
```

デザインやレイアウトはフロントエンド側で自動適用されるため、データだけ渡せば完了する。

## スライド設計ガイド

- 1枚のスライドで1つのメッセージを伝える
- 箇条書きは最大6項目、各項目は1行から2行に収める
- タイトルスライドには論文タイトル、著者名、所属、発表年を含める
- 最終スライドにまとめと展望を記載する
- 各スライドの情報量は均等に配分する

## 品質基準

- 論文の主張を正確に反映する
- 推測による補完は行わない
- 専門用語はそのまま使用し、必要に応じて簡潔な説明を添える
- スライド間の論理的な流れを保つ
